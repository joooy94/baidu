数据文件未纳入Git版本控制（总计约7.1GB），需手动放置到 `data/原始数据/` 目录下。所有CSV文件遵循统一的原始格式：

```
列: Bid, TagIndex, DateDay, DateHour, T01, T02, ..., T60
含义: 批次ID, 点位编号, 日期(YYYY-MM-DD), 小时(0~23), 第1分钟值, 第2分钟值, ..., 第60分钟值
```

## 算法设计

### 1. 传感器智慧诊断

三种异常检测，每种分三个报警等级（1=最高，3=最低）：

#### 1.1 数据缺失检测（`detect_data_absence`）

**原理：** 检测连续无效/缺失数据的时段长度。

| 连续缺失时长 | 报警等级 | 说明 |
|-------------|---------|------|
| ≥10分钟 | 三级 | 数据中断较短，影响有限 |
| ≥30分钟 | 二级 | 数据中断较长，需关注 |
| ≥60分钟 | 一级 | 数据严重中断，需立即处理 |

**算法步骤：**
1. 按时间排序，标记每条记录 `is_missing = ~is_valid`
2. 使用 `cumsum` 分组法识别连续缺失段
3. 计算每段持续时长，匹配最高报警等级

#### 1.2 数据漂移检测（`detect_data_drift`）

**原理：** 检测传感器读数相对于历史滑动均值的持续偏差。

| 传感器类型 | 三级 | 二级 | 一级 | 模式 |
|-----------|------|------|------|------|
| 电磁流量计 | ±20%/10min | ±40%/5min | ±50%/3min | 百分比 |
| 出水浊度仪 | ±15%/10min | ±40%/5min | ±50%/3min | 百分比 |
| 进水浊度仪 | ±15%/10min | ±40%/5min | ±50%/3min | 百分比 |
| 储液池液位计 | — | >10cm/1min | — | 绝对值(cm) |

**算法步骤：**
1. 使用30分钟滑动窗口计算历史均值 `hist_mean`
2. 计算百分比偏差 `(value - hist_mean) / hist_mean` 或绝对偏差
3. 找出连续超限段（按索引连续性分组），检查持续时长
4. 按阈值匹配报警等级

#### 1.3 数据异跳检测（`detect_data_jump`）

**原理：** 检测分钟级变化率异常。

| 传感器类型 | 判定条件 | 模式 |
|-----------|---------|------|
| 出水浊度仪 | 变化率 > 50 NTU/min，连续≥3个采样周期 | 固定阈值 |
| 进水浊度仪 | 变化率 > 正常波动3倍（3σ） | 正常波动倍数 |
| 电磁流量计 | 变化率 > 正常波动3倍（3σ） | 正常波动倍数 |
| 储液池液位计 | 10分钟内液位变化 > 30cm | 时间窗口 |

---

### 2. 投加设备健康度评估

**原理：** 从四个维度加权评估每台投矾泵（P1~P5）的健康状况，输出0~100综合评分。

#### 四维指标

| 指标 | 权重 | 计算方法 | 评分规则 |
|------|------|---------|---------|
| **流量偏差率** | 30% | `\|actual - rolling_mean\| / rolling_mean` | <5%→100分, 5~10%→80~100, 10~20%→60~80, 20~50%→30~60, >50%→0~30 |
| **故障频率** | 30% | 故障信号(error=1)出现频次/天 | 0次→100分, <1次→80~100, 1~5次→50~80, >5次→0~50 |
| **运行稳定性** | 20% | 变异系数 CV = std/mean | <0.05→100分, 0.05~0.1→80~100, 0.1~0.3→50~80, >0.3→0~50 |
| **控制响应性** | 20% | 自动(auto)与远控(remote)状态一致率 | >95%→100分, 80~95%→70~100, <80%→0~70 |

#### 等级划分

| 分数范围 | 等级 |
|---------|------|
| 90~100 | 优秀 |
| 70~90 | 良好 |
| 50~70 | 轻微异常 |
| 0~50 | 异常 |

#### 评分流程

```
原始数据 → 流量/故障/自动/远控四路信号
    │
    ├──→ _calc_flow_deviation()   → 滑动均值偏差率 → 线性映射分数
    ├──→ _calc_fault_frequency()  → 日均故障次数   → 线性映射分数
    ├──→ _calc_stability()        → CV变异系数     → 线性映射分数
    └──→ _calc_responsiveness()   → 状态一致率     → 线性映射分数
    │
    └──→ 加权求和 → 综合健康度 → 等级判定
```

### 3. 设备级智慧诊断

覆盖五大诊断类别：

| 诊断类别 | 检测项 | 报警等级 |
|---------|--------|---------|
| **投加泵异常** | 故障信号(error=1) | 二级 |
| | 自动模式运行但流量为0（≥5min） | 一级 |
| | 自动/远控状态不一致（≥10min） | 三级 |
| **管道堵塞/泄漏** | 泵有输出但管道流量接近0（≥5min） | 二级 |
| | 储液池液位异常快速下降（>0.5cm/min, ≥10min） | 一级 |
| **阀门异常** | 故障汇总信号=1 | 二级 |
| | 既未开到位也未关到位（≥3min） | 三级 |
| | 开到位和关到位同时为1 | 二级 |
| **储液池异常** | 液位>3.0m（≥5min） | 二级 |
| | 液位<0.3m / <0.1m | 二级/一级 |
| | 液位变化速率>0.3m/min（≥5min） | 三级 |
| **PLC通讯中断** | 指定时间范围完全无数据 | 一级 |
| | 数据覆盖率<90% | 二级 |

## 实验结果

使用2025年7月1日~7日的实际生产数据进行验证。

### 健康度评估结果

| 泵编号 | 综合评分 | 等级 | 流量偏差率 | 故障频率 | 运行稳定性 | 控制响应性 |
|-------|---------|------|-----------|---------|-----------|-----------|
| P1 | 52.0 | 轻微异常 | 30.6 (49.4%) | 100.0 (0次) | 44.0 (CV=0.38) | 19.9 (22.8%) |
| P2 | 59.7 | 轻微异常 | 53.2 (26.8%) | 100.0 (0次) | 48.7 (CV=0.32) | 19.9 (22.8%) |
| P3 | 53.3 | 轻微异常 | 33.2 (46.8%) | 100.0 (0次) | 46.6 (CV=0.35) | 20.0 (22.9%) |
| P4 | 52.3 | 轻微异常 | 31.7 (48.3%) | 100.0 (0次) | 43.7 (CV=0.39) | 19.9 (22.8%) |
| P5 | 41.0 | 异常 | 0.0 (130.5%) | 100.0 (0次) | 35.1 (CV=0.51) | 19.9 (22.8%) |

**分析：**
- 五台泵**故障频率均为满分**，7天内无故障发生
- **控制响应性普遍偏低**（~20%），auto与remote状态一致率仅22.8%，建议排查PLC控制逻辑
- **P5综合评分最低**（异常），流量偏差率高达130.5%，建议重点维护
- **P2表现相对最优**（59.7分），但运行稳定性和控制响应性仍需改善

- 浊度仪异常以**三级异跳为主**（占比93%），大部分为轻微波动，一级/二级严重异常极少
- 储液池液位计和电磁流量计在此期间数据正常
